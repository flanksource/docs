# <ConfigDb/> Catalog

ConfigDB component lookup enables you to query configs from the database to use as components.

```yaml title="kubernetes-cluster.yml"
apiVersion: canaries.flanksource.com/v1
kind: Topology
metadata:
  name: kubernetes-cluster
spec:
  type: KubernetesCluster
  icon: kubernetes
  schedule: '@every 10m'
  id:
    javascript: properties.id
  components:
    - name: nodes
      icon: server
      type: KubernetesNode
      // highlight-start
      lookup:
        catalog:
          - selector:
             - types:
                - Kubernetes::Node
      // highlight-end
```

| Field            | Description                                                                      | Scheme                                            | Required |
| ---------------- | -------------------------------------------------------------------------------- | ------------------------------------------------- | -------- |
| `display`        | Template to display query results in text (overrides default bar format for UI)  | [_Template_](../concepts/templating)           |          |
| **`selector`**      | Query that needs to be executed on the server                                    | [Resource Selector](/reference/resource-selector)                                          | Yes      |
| `transform`      | Template to transform results to                                                 | [_Template_](../concepts/templating)           |          |

## Results

The `results` variable will be a list of [Config][/reference/config-db/config-item]
