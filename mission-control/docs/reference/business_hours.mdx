Use business hours to evaluate whether a timestamp falls inside your configured working schedule.

The `in_business_hours` function is available in:

- [CEL expressions](/docs/reference/scripting/cel)
- [Go templates](/docs/reference/scripting/gotemplate)

Mission Control reads business hours from the `time_interval.business_hours` property.

## Default schedule

If you do not configure business hours, Mission Control uses:

- Monday through Friday
- 09:00 to 17:00

## Configure business hours

Set `time_interval.business_hours` in your properties file as a JSON array of time intervals.

```properties title="app.properties"
time_interval.business_hours=[{"weekdays":["monday:friday"],"times":[{"start_time":"09:00","end_time":"17:00"}],"location":"UTC"}]
```

The interval format supports these fields:

| Field           | Description                                                                           | Example                                       |
| --------------- | ------------------------------------------------------------------------------------- | --------------------------------------------- |
| `times`         | Time ranges in 24-hour format. `start_time` is inclusive and `end_time` is exclusive. | `[{"start_time":"09:00","end_time":"17:00"}]` |
| `weekdays`      | Weekday names or weekday ranges.                                                      | `["monday:friday"]`                           |
| `days_of_month` | Day-of-month values or ranges.                                                        | `["1:5", "-1"]`                               |
| `months`        | Month names or ranges.                                                                | `["january:march"]`                           |
| `years`         | Year values or ranges.                                                                | `["2025:2026"]`                               |
| `location`      | IANA timezone used for evaluation.                                                    | `"Asia/Kathmandu"`                            |

## Use in CEL expressions

`in_business_hours(<timestamp>)` returns `true` when the timestamp is inside business hours, and `false` otherwise.

```javascript title="CEL examples"
in_business_hours('2024-01-02T10:00:00Z') // true
in_business_hours('2024-01-02T20:00:00Z') // false
in_business_hours('2024-01-02 10:00:00') // true
```

## Use in Go templates

Use the same function in Go templates.

```go title="Go template examples"
{{ in_business_hours "2024-01-02T10:00:00Z" }} // true
{{ in_business_hours "2024-01-02T20:00:00Z" }} // false
```

## Timestamp formats

`in_business_hours` accepts these common formats:

- RFC3339, for example `2024-01-02T10:00:00Z`
- RFC3339Nano
- `YYYY-MM-DD HH:MM:SS`
- `YYYY-MM-DD`

If parsing fails, the expression returns an error.
