PostgreSQL connections enable integration with PostgreSQL databases for health checks, SQL queries in playbooks, and database monitoring.

## Used By

- [Postgres Health Check](/docs/reference/canary-checker) - Verify database connectivity and run health queries
- [SQL Action](/docs/guide/playbooks/actions/sql) - Execute SQL queries in playbooks
- [Postgres Scraper](/docs/guide/config-db/scrapers/postgres) - Scrape database configurations

## Example

```yaml title="postgres-connection.yaml"
apiVersion: mission-control.flanksource.com/v1
kind: Connection
metadata:
  name: payments-database
  namespace: default
spec:
  postgres:
    host:
      value: postgres.example.com
    port: 5432
    database:
      value: payments
    username:
      valueFrom:
        secretKeyRef:
          name: postgres-credentials
          key: POSTGRES_USER
    password:
      valueFrom:
        secretKeyRef:
          name: postgres-credentials
          key: POSTGRES_PASSWORD
    insecureTLS: false
```
