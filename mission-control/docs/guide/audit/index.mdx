# Audit & Compliance

Mission Control provides the building blocks for audit-ready infrastructure reporting. Config scrapers pull identity data, infrastructure state, and access logs from your cloud providers, identity systems, and databases into a unified catalog. The [Application CRD](/docs/guide/audit/applications) ties it all together into a single auditable unit per application - mapping access control, backups, environments, and compliance findings into one view.

## How It Works

<AuditDiagram className="mx-auto" />

**Scrapers** continuously pull data from three categories of sources:

- **Identity Providers** (Entra ID, AWS IAM, Kubernetes RBAC) - users, groups, roles, and access assignments
- **Infrastructure** (AWS, Azure, GCP, Kubernetes) - resources, configurations, and health status
- **Data Sources** (databases, CloudTrail, audit logs) - access logs, change events, and compliance metadata

All data flows into a **Unified Catalog** that stores users, groups, roles, access records, configuration changes, and infrastructure state in a single Postgres database.

From the catalog, three outputs serve audit and compliance needs:

- **[Application CRD](/docs/guide/audit/applications)** - maps catalog data to a specific application, surfacing access control, backup status, and environment locations in one view
- **[Views](/docs/guide/views/)** - custom dashboards for inventory, change trails, and access summaries
- **[Notifications](/docs/guide/notifications/)** - alerts on security-sensitive changes, access anomalies, and backup failures

## Concepts

- [Applications](/docs/guide/audit/applications) - The `Application` CRD ties infrastructure, access control, backups, and environments together into a single auditable unit
- [Access Control](/docs/guide/audit/access-control) - Track who has access to which resources via IAM, RBAC, and database roles
- [Backup Monitoring](/docs/guide/audit/backups) - Verify backup health, location, retention, and restore capability
- [Change Tracking](/docs/guide/audit/change-tracking) - Audit trail of every config change via diffs, CloudTrail, and Kubernetes events
- [Infrastructure Inventory](/docs/guide/audit/inventory) - Complete catalog of all resources with health status
