Mission Control integrates with AWS to provide visibility into your cloud infrastructure, monitor resource health, and automate remediation workflows.

---

<CapabilityHeading type="scraper" />

Discover and catalog AWS resources with full change history and drift detection.

**Use cases:**
- Track EC2 instances, RDS databases, and Lambda functions across accounts and regions
- Monitor S3 bucket policies and IAM configurations for compliance
- Correlate AWS resources with Kubernetes workloads and Terraform state
- Detect configuration drift and unauthorized changes

  <Card title="AWS Scraper Reference" link="/docs/guide/config-db/scrapers/aws" />
</Cards>

---

<CapabilityHeading type="healthcheck" />

Monitor AWS services and endpoints with automated health checks.

**Use cases:**
- Verify S3 bucket availability and object integrity
- Check CloudWatch metrics and alarms status
- Monitor SQS queue depths and message latency
- Test Lambda function invocations and response times

<Cards>
  <Card title="S3 Health Check" link="/docs/guide/canary-checker/reference/folder#s3" />

</Cards>

---

<CapabilityHeading type="playbook" />

Automate AWS operations and incident remediation with playbooks.

**Use cases:**
- Auto-restart failed EC2 instances or ECS tasks
- Scale Auto Scaling Groups based on custom metrics
- Rotate secrets and update SSM parameters
- Trigger AWS Lambda functions for custom remediation

<Cards>
  <Card title="Exec Action" link="/docs/guide/playbooks/actions/exec" />
</Cards>

---

<CapabilityHeading type="notifications" />

Send alerts to AWS notification services.

**Use cases:**
- Publish health check failures to SNS topics
- Create CloudWatch Events for config changes
- Integrate with AWS Incident Manager workflows
- Trigger Step Functions for complex alerting pipelines
