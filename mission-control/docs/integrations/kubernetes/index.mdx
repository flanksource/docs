Mission Control provides deep integration with Kubernetes clusters for complete visibility into your containerized infrastructure.

---

<CapabilityHeading type="scraper" />

Discover and catalog all Kubernetes resources with full change history and drift detection.

**Use cases:**
- Discover Deployments, Services, ConfigMaps, Secrets, and all Kubernetes resources
- Track configuration changes with full diff history
- Correlate workloads with Helm releases, GitOps deployments, and cloud infrastructure
- Monitor resource relationships and dependencies across namespaces

<CapabilityHeading type="healthcheck" />

Monitor pod health, resource usage, and cluster state with automated health checks.

**Use cases:**
- Verify pod readiness and liveness probe status
- Monitor resource usage against requests and limits
- Check cluster events for warnings and errors
- Validate service endpoints and ingress configurations

  <Card title="Kubernetes Health Check" link="/docs/guide/canary-checker/reference/kubernetes" />

<CapabilityHeading type="playbook" />

Automate Kubernetes operations and incident remediation with playbooks.

**Use cases:**
- Restart pods or rollout deployments automatically
- Scale deployments based on custom metrics or alerts
- Execute kubectl commands for debugging and remediation
- Delete stuck resources or force reconciliation

  <Card title="Kubernetes Action" link="/docs/guide/playbooks/actions/kubernetes" />

<CapabilityHeading type="notifications" />

Send alerts when Kubernetes resources change state or encounter issues.

**Use cases:**
- Alert when pods enter CrashLoopBackOff or ImagePullBackOff
- Notify on deployment rollout failures
- Track configuration changes to sensitive resources
- Integrate with Slack, Teams, or PagerDuty for on-call escalation
