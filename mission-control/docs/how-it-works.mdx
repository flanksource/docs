## Terms


<dl>
 <dt>Config Item</dt>
  <dd>A physical item with some underlying configuration e.g. `Pod`, `VM`, `File`, `Database`</dd>
 <dt>Changes</dt>
  <dd>Changes in the underlying configuration of a config item, or external events that occur on them </dd>
 <dt>Insights</dt>
  <dd>Recommendations and analysis provided by third parties through scanning and AI </dd>
 <dt>Relationships</dt>
  <dd>Links between config items based upon how they are logically created and categorized</dd>

 <dt>Component</dt>
<dd>A logical thing that often corresponds 1-1 with a config item, but not always e.g. `Pod`, `Finance Application`</dd>

  <dt>Health</dt>
    <dd>The RAG status of a component or config item e.g. <Healthy>Healthy</Healthy>, <Unhealthy>Unhealthy</Unhealthy>, <Warning>Warning</Warning></dd>
  <dt>Scrape</dt>
    <dd>Connecting to an external API and ingesting data from it</dd>
  <dt>Playbook</dt>
  <dd>A sequence of automated steps performed on a config item or component</dd>
  <dt>Canary</dt>
    <dd>A definition for how to perform a health check, Canaries produce 1 or more checks</dd>

    <dt>Check</dt>


  <dd>The output of a Canary (Health Check) that includes Health, Latency and other runtime details</dd>

## Kubernetes & Gitops

Mission control is kubernetes-native with all configuration being possible by Custom Resource Definition (CRD's)

The single source of truth is still the database, the operators only function is to synchronize CRD's into the database and update the CRD status periodically.





<Screenshot img="/img/architecture-flow.svg" shadow={false}/>




Mission Control uses Kubernetes Custom Resources Definitions as the underlying configuration method. The CRD's include:


| Resource                                                                | Description                                                                                     | Output |
| ----------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- | ------ |
| <Icon className="font-bold" name="k8s-customresourcedefinition" size={22}>ScrapeConfig</Icon> | Configures scrapers to run periodically | Config items, analysis, change, cost       |
| <Icon className="font-bold" name="k8s-customresourcedefinition">Canary</Icon>                 | Runs health checks using HTTP, SQL, S3, kubectl, etc.                                           | 1 or more checks (Health)        |
| <Icon className="font-bold" name="k8s-customresourcedefinition">Notification</Icon>           | Sends notifications based on events | Email, Slack, etc       |
| <Icon className="font-bold" name="k8s-customresourcedefinition">Playbook</Icon>               | Executable playbooks  | Pull Request, CLI,  Webhook        |
 <Icon className="font-bold" name="k8s-customresourcedefinition">Connection</Icon>             | Re-usable settings with secure external lookups | URL, username, password, etc        |
  <Icon className="font-bold" name="k8s-customresourcedefinition">Topology</Icon>             | Combines configs, health checks and lookups to form cards  | Component       |
